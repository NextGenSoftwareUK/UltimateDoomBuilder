To use real Quake 1 sprites for OQUAKE assets in the editor:
1. Get Quake 1 game data (id1 folder with pak0.pak), e.g. from Steam or GOG.
2. Build the ExtractOquakeSprites tool (in UDB Source\Tools\ExtractOquakeSprites).
3. Run the extractor and point output to the Sprites folder that UDB uses:
   - If you run UDB from the Build folder, use: Build\UDBScript\Scripts\OASIS\Sprites
   - Or use Assets so it is copied on next build: Assets\Common\UDBScript\Scripts\OASIS\Sprites
   Example:
     ExtractOquakeSprites.exe "C:\Source\vkQuake\id1" "C:\Source\UltimateDoomBuilder\Build\UDBScript\Scripts\OASIS\Sprites"
   Or for Assets:
     ExtractOquakeSprites.exe "C:\Source\vkQuake\id1" "C:\Source\UltimateDoomBuilder\Assets\Common\UDBScript\Scripts\OASIS\Sprites"
4. Restart UDB (or reload the map). OQUAKE things (types 5005, 5013, 5303, etc.)
   will show the Quake sprite; ODOOM things (5, 13, 2001, etc.) use game config.

Output: 5005.png = Gold key, 5013.png = Silver key, 5303.png = Shambler, etc.
If you don't have pak0.pak, you can add PNGs manually (name by unique type).
